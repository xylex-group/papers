\section{Executive Summary}

This document defines the runtime and architectural foundations of \textbf{Athena}, a Rust-based database gateway and driver system developed by XYLEX GROUP.

The core conclusion is:

\begin{quote}
\textbf{All database I/O in Athena must be built on Tokio with non-blocking async.}
\end{quote}

Rayon is retained only for CPU-bound post-processing. Blocking execution models are explicitly excluded from the core architecture.

Athena is not a traditional driver. It is a:

\begin{itemize}
\item multi-backend database gateway
\item routing layer with per-request client resolution
\item operational control plane (backups, provisioning, metrics)
\item unified query interface across PostgreSQL, ScyllaDB, Supabase
\end{itemize}

The system is designed for high concurrency, observability, and operational control.

\section{System Overview}

Athena exposes a unified API surface that routes requests dynamically to multiple backend databases.

\subsection{Core Capabilities}

\begin{itemize}
\item Gateway CRUD operations (\texttt{/gateway/*})
\item Multi-backend routing (Postgres, Scylla, Supabase)
\item In-memory client registry with hot reload
\item Deferred query queue
\item Admission control and rate limiting
\item In-process + Redis caching
\item API key system with granular rights

\section{Runtime Model Analysis}

\subsection{Rejected Model: Blocking + Rayon}

\begin{itemize}
\item Uses \texttt{spawn\_blocking} for I/O
\item Consumes OS threads per request
\item Limited scalability
\item Poor cancellation semantics
\end{itemize}

\textbf{Conclusion: Not suitable for Athena core.}

\subsection{Selected Model: Tokio Async}

\begin{itemize}
\item Event-driven I/O
\item Minimal threads
\item High concurrency
\item Native streaming + backpressure
\end{itemize}

\textbf{Conclusion: Required for all gateway operations.}

\subsection{Rayon Role}

Rayon is used only for:

\begin{itemize}
\item aggregation (group\_by)
\item transformation pipelines
\item heavy JSON processing
\end{itemize}

Never used for:

\begin{itemize}
\item database I/O
\item connection handling
\item request routing
\end{itemize}

\section{Conclusion}

Athena is architected as a high-performance database gateway built around modern Rust async principles.

The fundamental constraints are:

\begin{itemize}
\item Database workloads are I/O bound
\item Blocking models do not scale
\item Async execution is required for correctness under load
\end{itemize}

The resulting architecture is:

\begin{itemize}
\item Tokio for all I/O
\item Rayon for isolated CPU workloads
\item Strong separation between transport and processing
\end{itemize}

This design ensures Athena remains scalable, predictable, and maintainable under real-world load.
